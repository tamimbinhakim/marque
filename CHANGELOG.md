# Changelog

All notable changes to the Marque specification will be documented here. Changes to normative behavior are anchored to the MIP that introduced them.

The format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/). Version numbers follow `YYYY.MM.seq` where `seq` is the within-month revision counter.

Entries below this banner are generated by [`git-cliff`](https://git-cliff.org/) from [Conventional Commits](https://www.conventionalcommits.org/) (`<type>(<scope>): <subject>`). Do not hand-edit generated sections — edit the commit message and regenerate with `git-cliff --tag <next-tag> --prepend CHANGELOG.md`. See [CONTRIBUTING.md](./CONTRIBUTING.md#conventional-commits) for the commit convention and [`cliff.toml`](./cliff.toml) for the generator configuration.

## Version policy

- **Pre-charter** (this phase): each MIP merge bumps the revision counter. The spec is marked `pre-RFC` throughout. Release sections of this file are produced by `git-cliff` from the git log.
- **Post-IETF charter**: version tags align with Internet-Draft revisions (`draft-ietf-mpc-core-NN`).
- **Post-RFC publication**: the spec becomes immutable per IETF process; errata tracked in `docs/errata.md`.

Install once per developer: `brew install git-cliff` (macOS), `cargo install git-cliff` (any platform), or a release binary from [git-cliff.org](https://git-cliff.org/docs/installation).
